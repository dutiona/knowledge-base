# KB ‚áÑ ME Cross-Pollination ‚Äî Roadmap Parity at the Designed-Feature Level

## 1. Reframe (BLUF)

The previous take was anchored on current code and ran one direction only (KB‚ÜíME), which is why it dead-ended at "mostly already built." That framing is wrong twice over: it collapsed three independent axes into one, and it asked the wrong question. The correct frame is **roadmap parity at the designed-feature level** ‚Äî "which operators should each layer claim, when it's a good idea" ‚Äî driven by a canvas of 61 operators (notes 34/35), kept **bidirectional** (KB‚ÜîME), and decoupled into three axes that move independently: **(1) operators (the math), (2) data structures each operator needs, (3) what is in code now**. Axis 3 is the _last_ thing to consult, as gap analysis ‚Äî never the framing. The single biggest move: **adopt the use-now faithful-metric ground layer (anisotropy-correction + CSLS, then diffusion-distance + LID) and pin it as a SHARED cross-layer contract ‚Äî proposed ADR-0017 ‚Äî because KB and ME embed with the same model (ADR-0015) and therefore inherit bit-for-bit the same anisotropy, so the de-biasing must be solved once, in parity, not twice.** For KB specifically this is not memory machinery ‚Äî it is pure RAG retrieval-quality gold the previous pass dried out.

Two cross-cutting discipline checks the owner asked us to hold (and which the final report keeps explicit, not implicit):

- **No refuted premise is load-bearing anywhere.** Relation-as-offset / analogy arithmetic stays quarantined in ¬ß8 (~9‚Äì11% BATS, superseded by parallel-transport + GW), and nothing recommended below rests on it. The shared-vs-layer boundary in ¬ß8 is a clean, single-sentence contract.
- **The three axes never collapse back into "current code."** Every operator below is presented as math ‚Üí data structure ‚Üí status, with status read _last_. The previous pass's failure (reading "the data structure looks similar" as "the operator is built") is not repeated; new second-wave operators added in this pass (LEACE, GW, persistent-homology, Euler-œá) carry their axis-2 data structure explicitly, not a shortcut "new."

## 2. The three decoupled axes (kept separate, on purpose)

These three are orthogonal. The previous pass's error was reading "the data structure looks similar to something KB has" as "the operator is built." It isn't ‚Äî a `vec0` table is the raw active space, not a whitened space; `embed_spaces` is a registry, not a `projection_matrices` table; a daemon `Thread` is not a scheduler.

**Axis 1 ‚Äî OPERATORS (the math).** The note-34 canvas: 61 operators in 4 tiers (use-now / worth-exploring / could-work / probably-not), composed under one principle ‚Äî _build the faithful-metric ground layer (anisotropy + CSLS) first; every spectral/curvature/transport operator inherits its validity from de-biased edge weights._ An operator is a capability claim, independent of where its data lives or whether either repo has written it yet.

**Axis 2 ‚Äî DATA STRUCTURES (what each operator needs materialized).** Note 35's inventory, decoupled by compute tier (real-time / blocking / offline) and by freshness trigger (ingest / expire / consolidation / model-swap). The whitened/isotropic space + projection matrix `W` is the _universal prerequisite_ ‚Äî no de-biased operator is valid without it. Other structures: kNN similarity-edge graph, sparse CSR adjacency, Laplacian eigenbasis, diffusion-map coords, LID column, ORC/FRC cache, **LEACE concept-erasure projection**, **GW-coupling cache**, **persistence diagrams + Mapper graph**, dictionary atoms, plus the freshness machinery (`DerivedStructureRegistry`, `SpaceEpochCounter`, `IngestDirtyNotifier`, `MaterializationScheduler`). A structure can be _needed_ by an operator the repo hasn't adopted, and _present_ (as a registry shell) without the operator existing.

**Axis 3 ‚Äî WHAT'S IN CODE NOW (gap analysis, LAST).** KB: raw cosine everywhere ‚Äî `search.py` vec0, `folder_summaries.py` mean-pool centroid (1.15√ó boost), `auto_relate.py` 0.82 raw-cosine cutoff; zero whitening / CSLS / diffusion / LID / curvature / concept-erasure (grep over `src/` for `whiten|anisotrop|csls|diffusion|curvature|ricci|isotrop|leace|inlp` = all-zero, verified). ME: the geometric programme is a _tracked designed roadmap_ (epic #761 ‚Üí E0‚ÄìE5 sub-epics #763‚Äì#770) with the substrate seams present (`embedding_spaces`, `fact_vectors`, `config` epoch host, #623/#742 atomic promote) but the whitening/CSLS math itself unbuilt. Both layers are at axis-3 zero on the operator math; only the structural shells differ.

## 4. The shared cross-layer faithful-metric layer (proposed ADR-0017)

**What it is.** ADR-0017 is the metric-level sibling of ADR-0015. 0015 pins _which space a vector belongs to_ (the identity tuple `{model, provider, dim, matryoshka_base_dim, element_type}`); **0017 pins which faithful metric that space is read under.** A vector is only safely exchangeable between layers when _both_ are fixed.

**What both repos maintain in parity:**

- **The whitened/isotropic space** ‚Äî operator `faithful_sim(x,y) = CSLS_k(W(x‚àíŒº), W(y‚àíŒº))`; the _form_ (mean-center ‚Üí whiten ‚Üí CSLS) is normative, the fit params (`p`, `k`, ZCA-vs-ABT) recorded per space and **identical across layers for a shared model**.
- **`projection_matrices`** ‚Äî the normative home for `W`/`Œº`, field names fixed the way 0015's tuple fields are: `(space_id, raw_space_id, method, p, mu_blob, matrix_blob, csls_k, fitted_at, source_row_count, fingerprint)`.
- **The freshness/epoch invalidation rule** ‚Äî `SpaceEpochCounter` (bumped _inside_ the promote tx) + `DerivedStructureRegistry` carrying a **content fingerprint** = `hash{epoch, W_version, csls_k, row_count_bucket}` (epoch-alone is insufficient: retuning `W` without a model swap leaves epoch unchanged but the metric wrong ‚Äî "looks fresh, is corrupt").
- **Composition with 0015 + reconstruction (#623/#742)** ‚Äî `W` is undefined across identity tuples; a 0015 reconstruction invalidates `W` and all derived structures atomically; the epoch bump rides the same transaction. 0017 ¬ß4 is to 0015 ¬ß4 as a rider is to a contract.
- **Quantization ordering clause** ‚Äî int8/PQ quantization (KB‚ÜíME flow, ¬ß5) is applied **after** whitening, never before: quantizing the raw anisotropic space re-introduces the very anisotropy `W` removed. The contract fixes the order so a quantized vector exchanged between layers is still on the faithful metric.

**Why shared, not per-layer.** Anisotropy is a property of the _embedding model_, not the layer that hosts it. bge-m3 produces the same anisotropic geometry in KB's `vec0` tables and ME's `fact_vectors`. `W` and `Œº` are pure functions of (model, distribution); two stores over the same model require the _same_ `W` to be comparable. This same logic extends past `W`: any _fitted basis_ that is a function of the shared model's geometry ‚Äî LEACE concept-erasure axes, the dictionary atom basis, the Laplacian eigenbasis ‚Äî is also a model-property, so its **operator + fitted basis is parity-recommended** (only the per-layer point _assignments/labels_ are layer-local). See ¬ß8 for the exact split.

**The divergence risk (why the obligation must be pinned).** If KB and ME each fit `W` independently, a vector authored under `W_KB` and read under `W_ME` is compared across two different linear maps ‚Äî **not a metric at all**. `Œº_KB ‚âÝ Œº_ME` (papers vs. memories differ), so even mean-centering shifts them by different offsets; CSLS makes it worse (hub density `r_k` over different neighbor sets). This is _exactly_ the 0015 silent-corruption mode ‚Äî "numerically valid, semantically meaningless" ‚Äî relocated from identity to metric, and **invisible to 0015's mismatch check** because the tuples are _equal_ (`W` is not in the tuple). Cosine still returns a number in [‚àí1,1]; nothing throws; recall is just quietly wrong, and every downstream Laplacian/ORC/diffusion/LEACE/GW edge inherits the divergence. **Identity parity without metric parity is a false sense of safety.** For true co-hosting only **shared-fit** (one `W` blob loaded by both) is correct; **pinned-parameter** (same method/`p`/`k`, receiver always re-whitens) is a migration bridge with known residual `Œº`-offset error, not a clean guarantee.

## 6. Infra for the cloud / multi-machine end-state

The owner's end-state ‚Äî KB DB + ME DB in the cloud, shared across machines ‚Äî requires three infra epics in a **strict dependency chain: T1 ‚Üí T2 ‚Üí T3-lite, with T3-full deferred.**

**T1 ‚Äî Pluggable StorageBackend (mirror ME #628).** Domain code talks to a persistence port, never `sqlite3`. Six bounded-context Protocols aggregated by `KbStorageBackend`: `ChunkStore`, `VectorIndex`, `SearchIndex`, `PaperGraph`, `JobStore`, `SchemaManager`. Python adaptation: `typing.Protocol` (structural conformance, no `impl` boilerplate, one `@runtime_checkable` smoke assert at the composition root) + a **conformance battery parametrized over `[sqlite, pg]`** (the single most load-bearing artifact ‚Äî without it "pluggable" is a lie found in production). pgvector maps KB's per-space `vec0` to one `chunk_vectors(chunk_id, space_id, embedding VECTOR(D))` partitioned by `space_id` (verbatim ME's `fact_vectors`), HNSW server-side ANN (subsumes #65 LanceDB as a _third_ `VectorIndex` impl), FTS5‚Üí`tsvector`+GIN. **#65 re-scopes to "LanceDB VectorIndex impl" after the port; #124 build/serve decouple is a co-requisite** (a shared cloud DB makes the ingest/serve split mandatory, not optional ‚Äî and, per ¬ß5, that decouple is itself a KB‚ÜíME architectural contribution). **Feature-altitude decision to surface:** Postgres `ts_rank_cd` has no corpus IDF ‚Äî RRF tolerates it (rank-based), but `prediction_errors.py`'s absolute-score threshold (0.025, FTS5-calibrated) needs re-calibration or a BM25 extension. Accept ranker drift on cloud, or gate cloud on a BM25 extension? **This re-calibration is filed as a #473 sub-issue (¬ß7), not left as floating prose.**

**T2 ‚Äî Scoping (mirror ME hierarchical scoping).** `scopes(id, parent_id, label, depth)` tree + root sentinel + in-memory `ScopeTree` + `scope_id INTEGER NOT NULL DEFAULT 1 REFERENCES scopes(id)` on `chunks/papers/relationships/conclusions/folder_summaries`. `DEFAULT 1` makes the migration non-breaking. **Copy ME's fail-closed default verbatim: empty/defaulted scope resolves to _nothing_, not "everything"** (the guard test `resolve_query_empty_string_is_no_match_not_everything` is the tripwire against silent unscoped scans ‚Äî catastrophic for an isolation primitive). **Feature-altitude decision:** scope ‚ä• embedding space (option A, recommended ‚Äî preserves ADR-0015's single shared metric the geometric layer depends on) vs. scope-pinned spaces (option B ‚Äî fractures the shared-metric guarantee). Default A. Ties: **scope first ‚Üí #263 ACL second** (ACL is enforcement _on top of_ scope; scope alone delivers the single-owner multi-machine value, ACL is the multi-_tenant_ upgrade, correctly deferred); #264 `contributing_agent` is provenance (a column), kept separable from partition.

**T3 ‚Äî Async/queue/scheduler, SPLIT.** **Not a full asyncio rewrite** ‚Äî KB's bottleneck is Ollama + blocking C (pymupdf4llm, sqlite3); `async def` over blocking C is a thread pool in a coroutine costume. Mirror ME's _discipline_, not its tokio runtime.

- **T3-lite (now, with/after #124):** harden the existing `jobs` queue ‚Äî `worker_id`/`claimed_by` + lease/heartbeat (recover dead remote workers, not just crash-on-boot), retry/backoff (`attempts`, `next_retry_at`), priority, and behind the `JobStore` port a Postgres impl with `SELECT ‚Ä¶ FOR UPDATE SKIP LOCKED` + `LISTEN/NOTIFY` (replaces the 5s poll). Plus a **global Ollama-aware semaphore** so N parallel ingest jobs don't stampede one Ollama. This is what the cloud end-state needs.
- **T3-full (defer, gate on the geometric programme):** `DerivedStructureRegistry` + `SpaceEpochCounter` + `IngestDirtyNotifier` + `MaterializationScheduler` (a daemon polling `derived_structures` for `epoch_built < current_epoch`, enqueuing rebuilds at tier cadence on the existing queue). Building it before there's a single derived structure to materialize is speculative ‚Äî pull it forward _when KB adopts its first geometric operator_ (the whitening ground layer), which is exactly when whitening/diffusion/ORC/LEACE/GW caches start going stale on ingest.

**Prerequisite ordering:** #468/#469 (ADR-0015 parity) ‚Üí T1 (port + pgvector + #124) ‚Üí T2 (scoping) and T3-lite in parallel ‚Üí **minimal cloud-shared end-state**. T3-full + #263 ACL are the next tier. T3-full and the geometric ground layer share the freshness machinery ‚Äî co-design them.
