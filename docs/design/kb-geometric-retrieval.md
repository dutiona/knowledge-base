## 1. Where KB sits today, and the one thing that gates everything

**BLUF.** KB applies **zero** de-biased, spectral, curvature, transport, or concept-erasure operators today. The entire retrieval stack ‚Äî FTS5 BM25 + sqlite-vec cosine fused by Reciprocal Rank Fusion (RRF), the cross-encoder reranker, the folder-context boost, and auto-relationship discovery ‚Äî sits on **raw cosine over the un-de-biased anisotropic embedding cone**. The first thing to build is the **faithful-metric gate** (anisotropy-correction / whitening + CSLS hubness correction). Everything else in this program inherits its validity from de-biased edge weights; building any spectral/curvature/transport operator before the gate exists silently corrupts all of them.

The verified anisotropy claim is **Ethayarajh (2019)** ‚Äî contextual embeddings of the BERT/Qwen/bge family occupy a narrow anisotropic cone; mean random-pair cosine sits far above zero. KB embeds with **bge-m3 via Ollama** today (Qwen possible later). When KB and ME are co-hosted they embed with the **same model**, so they inherit the **same anisotropy** ‚Äî the correction is not a per-layer accident, it is a shared property of the shared encoder, and that is precisely why ADR-0015 (embedding-identity) and ADR-0017 (metric) are cross-layer contracts.

What the gap looks like in concrete KB code (axis 3, the gap-analysis ‚Äî kept strictly separate, see ¬ß2):

| Site                        | Today                                                                                                                                                                     | Problem                                                                                                                                                                                                           |
| --------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `search.py` (hybrid search) | FTS5 BM25 ‚äï sqlite-vec **raw cosine** ‚Üí RRF; then `_folder_boost` multiplies by `FOLDER_BOOST_FACTOR = 1.15` (search.py:48) over a `BOOST_WINDOW_MULTIPLIER`-sized window | every vector rank is an anisotropy-biased rank; RRF fuses a biased list with BM25; the reranker (below) only re-sorts whatever this biased candidate set surfaced                                                 |
| `auto_relate.py`            | brute-force kNN, **raw cosine**, absolute cutoff `auto_relate_propose_threshold` default **0.82** (auto_relate.py:60)                                                     | an _absolute_ cosine cutoff on an anisotropic cone is the textbook hubness failure ‚Äî hub chunks clear 0.82 against everything, rare chunks never do; the cutoff means different things in dense vs sparse regions |
| `folder_summaries.py`       | folder centroid = **mean-pool** of member-chunk vectors; consumed by the search-time boost                                                                                | a Euclidean mean of anisotropic vectors lands inside the cone, biased toward the dominant direction; the boost amplifies that bias                                                                                |
| `reranker.py`               | cross-encoder over the top candidates                                                                                                                                     | **downstream of the metric** ‚Äî it cannot recover a relevant chunk the biased first stage never retrieved                                                                                                          |

The reranker is the tell: KB already spent a cross-encoder to paper over first-stage ranking quality. That is the right instinct aimed one layer too late. **De-bias the metric, and the reranker gets a better candidate set for free.**

**The composition principle (note 34 ¬ß6, restated for RAG):** build the faithful-metric ground layer **first**, because every spectral, curvature, transport, and structural operator below inherits its validity from de-biased edge weights. Skipping it silently corrupts the Laplacian, the curvature, and the transport plans downstream ‚Äî and in RAG terms, corrupts precision\@k, the auto-relate graph, and the corpus-health monitors all at once.

## 2. Three decoupled axes ‚Äî kept explicitly separate

This document obeys the owner framing: **lead by the ideal**, and decouple three axes that are routinely conflated. They are reasoned about in this order and never collapsed.

- **Axis 1 ‚Äî Operators (the math).** What mathematical maps are _correct and useful_ for RAG, independent of whether KB can run them today. Verdict tiers (`use-now` / `worth-exploring` / `could-work` / `refuted`) come from note 34's adversarial verification. This is ¬ß3.
- **Axis 2 ‚Äî Data structures (what each operator needs).** For each operator, the coordinate space, cache, or graph it must read from ‚Äî partitioned by **compute tier** (real-time / blocking / offline) because the tier, not the operator, decides where it can live in a request. This is ¬ß4. Crucially, an operator and its structure are separable: the same whitened space serves a dozen operators; the same diffusion ranker can be fed by Nystr√∂m landmarks or a full eigenbasis.
- **Axis 3 ‚Äî What's in code now (gap analysis, LAST).** The honest `have / extend / new` accounting against KB's actual schema (`chunks`, the per-space `vec0` tables, `embed_spaces`, `relationships`, `config`) and modules. This is the _last_ axis, used only for gap-analysis and sequencing ‚Äî never to prune the ideal. It appears as the rightmost column of every table and as the explicit sequencing in ¬ß5‚Äì¬ß6.

The discipline this enforces: a structure being `new` (axis 3) is **never** a reason to drop an operator that is `use-now` (axis 1). It is only an input to _ordering_. KB ships almost nothing geometric today; if axis 3 drove the design, the design would be empty.

**RAG scope (what KB cares about, vs what it does not).** KB is the **Knowledge layer** ‚Äî a retrieval system. Its concerns are: retrieval **precision/recall**, **de-biased similarity**, **hubness**, **query understanding**, **novelty / abstention** (knowing when the corpus has no answer), **corpus clustering / routing**, **cross-document contradiction & transfer**, and **dedup**. KB does **not** own memory _dynamics_ ‚Äî forgetting curves, decay, attractor recall, temporal point processes, trajectory forecasting. Those are ME's (the Memory layer). Operators that are purely dynamical (Modern Hopfield completion, Hawkes self-exciting access, Ebbinghaus/forgetting, Koopman/DMD forecasting, Kalman tracking) are **ME-only and excluded here** ‚Äî they are named where relevant so the boundary is explicit, then dropped. The shared substrate (whitened space, CSLS, the identity tuple) is shared; the dynamics are not. ¬ß7 draws the line crisply.

### 3.3 Memory-dynamics operators ‚Äî **ME-only, excluded from KB**

Named once for an explicit boundary, then dropped. These read a _temporal/scalar_ field over memories (decay, access bursts, trajectories) ‚Äî they are dynamics, not retrieval. **KB does not build them.**

- **Modern Hopfield / attractor completion** (Ramsauer et al. 2021) ‚Äî content-addressable recall _dynamics_; ME's recall epic, not KB.
- **Hawkes self-exciting point processes** (Hawkes 1971; Ozaki 1979) ‚Äî bursty access intensity for forgetting; ME E4-temporal.
- **Ebbinghaus / forgetting curves** ‚Äî decay dynamics; ME's core, KB has no decay.
- **Koopman / DMD, Kalman / SSM** (Brunton et al. 2022; Schmid 2010) ‚Äî trajectory _forecasting_; ME prediction. KB retrieval is stateless w.r.t. time.
- **VSA / HRR typed-edge binding** (Plate 1995) ‚Äî within-space typed-edge _traversal_; ME's graph algebra. KB's `relationships` table is small and SQL-traversed; no VSA need.

### 3.4 Refuted ‚Äî quarantined, do not build

- **Relation-as-offset / analogy arithmetic** ("king‚àíman+woman"). **Refuted for abstract relations:** ~9‚Äì11% on lexicographic BATS, degraded further on anisotropic embeddings. The geometric reason: cosine-offset is a _flat, zero-curvature_ tool; on a curved anisotropic manifold the relation does not translate. **Do not rebuild offset arithmetic.** Route any cross-document transfer need through **GW** (coordinate-free structure) or **LEACE** (strip-then-compare) instead.

## 4. Data-structure inventory (KB) ‚Äî by compute tier

The operational core. Each structure: what operators it supports, when it is built, its freshness trigger, the `have/extend/new` verdict against KB's actual schema, and size/cost. **Compute tier is the organizing axis** because it decides where a structure can live relative to a request ‚Äî a real-time structure can be read inside `search_index`; a blocking one must be off-path (a job in `jobs.py`); an offline one runs only in a consolidation/reindex sweep.

KB's append-only-ish ingestion means **every derived spectral/curvature/diffusion artifact is invalidated as the corpus grows** ‚Äî the central engineering constraint, identical in shape to ME's (note 35 ¬ß1). The plumbing in ¬ß4.4 exists precisely to manage that.

**Why whitened is the gate, in RAG terms:** raw bge-m3 random-pair cosine sits far above zero. Every edge weight in the auto-relate kNN graph, every RRF vector rank, every folder centroid, every reranker candidate is anisotropy-biased high. The Laplacian built on those weights is biased; ORC/GW/diffusion built on it inherit the bias. **The whitened space is the precondition for correctness of the entire program ‚Äî not polish.** The `vec0` table machinery and `embed_swap.py` reconstruction pipeline already give KB the seam to register and backfill a derived space; the whitened space is mostly _plumbing reuse_ plus the `projection_matrices` table.

### 4.3 Freshness / invalidation matrix

Four triggers √ó maintenance strategy. RAG-scoped triggers: **ingest** (new chunk), **expire** (deletion/re-ingest of a source), **reindex/consolidation** (a sweep), **model-swap** (the `embed_swap.py` reconstruction path).

| Structure                                          | ingest                              | expire                | reindex/consolidation               | model-swap (`embed_swap`)                              |
| -------------------------------------------------- | ----------------------------------- | --------------------- | ----------------------------------- | ------------------------------------------------------ |
| `vec0` raw space (have)                            | append                              | tombstone             | ‚Äî                                   | reconstruct ‚Üí new space                                |
| Whitened space, anisotropy/CSLS cache              | dirty-flag                          | dirty-flag            | rebuild                             | **full-rebuild** (W meaningless at new model)          |
| kNN similarity-edges                               | incremental (top-k INSERTs)         | delete its edges      | optional full rebuild               | **wholesale delete+rebuild** (cosine changes globally) |
| LID, FRC, Euler-œá, RMT-k\*                         | **real-time-incremental**           | incremental decrement | bulk recompute stragglers           | full-rebuild (reset to NULL)                           |
| Laplacian eigenbasis, diffusion, spectral clusters | dirty-flag (drift OK until reindex) | dirty-flag            | **dirty-flag ‚Üí amortize in sweep**  | delete (epoch mismatch)                                |
| Dictionary atoms, cluster labels                   | online incremental                  | flag for refit        | recompute                           | full-rebuild (atoms meaningless at new dim)            |
| GW couplings, PH diagrams, Mapper, barycenters     | dirty-flag                          | dirty-flag            | **dirty-flag ‚Üí amortize** (offline) | full-rebuild                                           |

### 4.4 Plumbing ‚Äî the freshness machinery

Three new pieces of infrastructure that every cache above depends on:

- **`DerivedStructureRegistry`** ‚Äî one row per `(structure_kind √ó space)` carrying `dirty_since` / `status` / a **content fingerprint**. This is the scheduler's source of truth for what is stale.
- **`SpaceEpochCounter`** ‚Äî a `config` u64 row, **bumped inside the promote/reindex transaction** so any reader who sees the promoted vectors also sees the new epoch. (KB's `config` is already K/V, so this is one row, no migration.)
- **`IngestDirtyNotifier`** ‚Äî extends the ingest path (`ingest.py` ‚Üí `jobs.py`): on commit, flip registry dirty _and_ do inline real-time-tier compute (LID, FRC, œá increment).
- **`MaterializationScheduler`** ‚Äî a `jobs.py` sub-step that claims dirty rows (`status='rebuilding'`), runs the CPU-heavy build off-path, flips to `fresh`. Two mandatory startup consistency checks: re-dirty anything whose `last_computed_at < MAX(chunk created)` (crash between commit and notify), and `status='rebuilding' ‚Üí 'dirty'` (crash mid-build).

**Critical correctness rule ‚Äî epoch alone is insufficient (note 35 C2).** `SpaceEpochCounter` guards _embedding-space swaps_ only. But edge weights also change when the whitening matrix `W` is retuned, or the CSLS `k` is changed, **without** a promote ‚Äî epoch unchanged, yet the metric is now wrong: _"looks fresh, is corrupt"_ ‚Äî the single most dangerous failure mode. Therefore the registry's invalidation key is a **content fingerprint**:

```
fingerprint = hash{ space_epoch, W_version, csls_k, row_count_bucket }
```

Invalidate on **fingerprint mismatch**, never on epoch alone. This is a load-bearing line and goes verbatim into ADR-0017.

### 4.5 Mapping onto KB's actual schema

- `chunks` ‚Äî gains `lid` (REAL), `cluster_id` (INT). Both nullable, computed lazily; never blocks ingestion.
- per-space `vec0` tables ‚Äî unchanged for raw; **one new `vec0` per derived space** (whitened, diffusion-map), exactly the existing multi-space pattern.
- `embed_spaces` registry `{name, model, provider, dim, matryoshka_base_dim, element_type, status, table_name, chunk_count}` ‚Äî gains derived-space rows (whitened/diffusion). The `status` column already models `active/populating/deprecated`; derived spaces reuse it.
- `relationships` ‚Äî gains `similarity_knn` relation semantics (the proximity graph), `frc` (REAL per-edge), and the ORC cache references it. Auto-relate's output edges live here today; the geometric edges join them.
- `config` ‚Äî gains `space_epoch`, fingerprint components, RMT scalars, the Euler-œá series. All K/V, no migration.
- **new tables:** `projection_matrices` (`W, Œº, P` blobs keyed by space + version), `derived_structure_registry`, `dictionary_atoms` + inverted `atom_chunks`, `mapper_nodes`/`mapper_edges`, persistence-diagram blobs.

## 5. The KB program ‚Äî E0-analog ‚Üí downstream

The build order, leading by the ideal. KB's E0 is ME's E0: the faithful-metric substrate. Everything downstream gates on it.

### E0 ‚Äî Faithful-metric substrate (the gate)

- **Operators:** anisotropy-correction/whitening, CSLS, RMT rank guard, Euler-œá monitor.
- **Structures:** whitened `embed_spaces` row + `projection_matrices(W, Œº)`; CSLS `r_k` cache; RMT k\* cache; `DerivedStructureRegistry` + `SpaceEpochCounter` + fingerprint.
- **User-visible RAG payoff:** precision\@k rises across the board (de-biased vector rank); auto-relate's broken absolute 0.82 cutoff is replaced by a hubness-corrected score that means the same thing in dense and sparse regions; folder boost and reranker both operate on a de-biased candidate set. A `embed_swap` that produces a degenerate space is now _refused_ by the RMT guard.
- **Gating:** gated on the **identity parity** chain (#468 mismatch detection, #469 ADR-0015 mirror) ‚Äî KB must know _which_ model produced the vectors before it whitens them. Then **re-point `auto_relate`, `folder_summaries`, and `search` at the whitened space** ‚Äî this re-pointing is the moment the gate pays off; until then the whitened space is built but unused.

### E1 ‚Äî Recall upgrade (diffusion + LID)

- **Operators:** diffusion-distance ranker (on the whitened kNN graph), LID adaptive-`k`, OOD abstention.
- **Structures:** Laplacian eigenbasis + diffusion-map coords + Nystr√∂m landmarks; `chunks.lid`.
- **Payoff:** multi-hop recall (find chunks reachable through the corpus, not just cosine-near); the ranker stops being fooled by a single spurious anisotropy edge; **abstention** ‚Äî when LID says the query is in an empty region, KB returns "no confident answer" instead of a low-quality top-1 (a direct novelty/abstention win the owner named).
- **Gating:** gated on E0 (the eigenbasis must be built on whitened edge weights). LID adaptive-`k` is independently shippable on the whitened metric.

### E2 ‚Äî Concept structure (LEACE + dictionary + spectral routing)

- **Operators:** LEACE/INLP concept-erasure, dictionary-atom concept index, spectral clustering + query routing.
- **Structures:** LEACE `P`; dictionary `D`/`X` + inverted `atom‚Üíchunk` index; `chunks.cluster_id` + cluster centroids.
- **Payoff:** facet-invariant search (strip "is-recent" / "is-Python" to retrieve on substance); a concept-level retrieval handle (`atom‚Üíchunks`) that surfaces relatedness cosine misses; coarse-to-fine routing (route to a cluster, then retrieve) that cuts candidate count and lifts precision; **dedup** via shared-atom detection.
- **Gating:** LEACE and the dictionary gate on E0 (whitened space). Spectral routing gates on E1's eigenbasis.

### E3 ‚Äî Cross-document transfer & contradiction (GW + PH/GFT + curvature)

- **Operators:** GW cross-corpus alignment; persistent-homology contradiction cycles + GFT high-frequency contradiction; ORC/FRC bridge detection; Wasserstein-SW corpus-drift monitor; Wasserstein barycenter centroids.
- **Structures:** GW coupling cache; persistence diagrams + Mapper; ORC cache + FRC column; cost matrices + barycenters.
- **Payoff:** cross-document analogy retrieval (align two corpora with no shared coordinate); a contradiction detector feeding `prediction_errors` (chunks that disagree with their neighbors); a corpus-drift alarm wiring the dormant drift work; a _better folder centroid_ (barycenter, not mean-pool).
- **Gating:** all gate on E0; GW/PH are offline-only (capped region sizes). Curvature FRC is the cheapest and can land on the ingest hot path early.

### 6.1 Delta 14 ‚Äî provider abstraction (ADR-0018), in detail

Today KB has **two un-unified provider systems** (axis-3 fact): `embeddings.py` has an `EmbeddingProvider` Protocol with `OllamaProvider`, `OpenAIProvider` (**hardcoded** `https://api.openai.com/v1/embeddings`, _not_ `base_url`-configurable), `ONNXProvider`; `llm.py` has a _separate_ config-driven provider in `{ollama, openai_compat}` where `openai_compat` hits `{base_url}/v1/chat/completions` with a Bearer token and an SSRF guard ‚Äî but no Anthropic path. A clean provider abstraction that _carries the identity tuple_ is what lets KB and ME (co-hosted, same model, same anisotropy) share the metric contract. It belongs to **#474** (embedding/quantization) primarily, or **#480** if treated purely as infra.

## 7. Shared (ADR-0017) vs layer-specific boundary

The crisp line between what KB and ME **share verbatim** and what each owns.

**Shared ‚Äî ADR-0017 (cross-layer, mirrored verbatim in both repos), because co-hosted KB and ME embed with the same model and inherit the same anisotropy:**

- The **faithful-metric definition**: whitened/isotropic space as the metric ground truth; `metric = CSLS over W(v‚àíŒº)`. Both layers' similarity _means_ the same thing.
- The **embedding-identity tuple** (ADR-0015, already shared verbatim) `(model, provider, dim, matryoshka_base_dim, element_type)` ‚Äî the key everything geometric is parameterized by.
- The **content-fingerprint invalidation rule** `hash{epoch, W_version, csls_k, row_count_bucket}` and the "epoch-alone-is-insufficient" guarantee. A shared correctness invariant.
- The **operator math and verdict tiers** (notes 34/35) ‚Äî the same verified taxonomy grounds both programs; neither re-derives it.
- The **StorageBackend port shape** (KB delta 9 mirrors ME #628) so a vector lives in the same abstract place in both.

**Layer-specific ‚Äî each owns, never shared:**

|                        | KB (Knowledge / this doc)                                                                                                                        | ME (Memory / `geometric-associative-memory.md`)                                                           |
| ---------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------ | --------------------------------------------------------------------------------------------------------- |
| **Concern**            | retrieval precision/recall, hubness, query understanding, novelty/abstention, corpus routing, **cross-document** contradiction & transfer, dedup | associative recall, abstraction, **cross-memory** transfer, prediction, **forgetting**, pattern-detection |
| **Excluded operators** | (builds the static/structural operators)                                                                                                         | ‚Äî                                                                                                         |
| **Owns the dynamics**  | none ‚Äî KB is stateless w.r.t. time                                                                                                               | Hopfield, Hawkes, Ebbinghaus, Koopman/Kalman, VSA typed-edge traversal                                    |
| **Unit of geometry**   | the **corpus** (documents, chunks, folders)                                                                                                      | the **memory store** (facts, edges, bi-temporal snapshots)                                                |
| **Epic mapping**       | #471‚Üí#472/#473/#474/#477/#478/#480/#482                                                                                                          | #761‚Üí#763 (E0) /#765 (E1)/#767 (E2)/#768 (E3)/#769 (E4)/#770 (E5)                                         |
| **Build trigger**      | reindex / consolidation / `embed_swap`                                                                                                           | dream-cycle / consolidation / reconstruction                                                              |

**The one-sentence boundary:** _KB and ME share the faithful-metric substrate, the identity tuple, and the invalidation contract (ADR-0015 + ADR-0017); KB then builds the **static/structural** operators (de-biased similarity, clustering, routing, cross-document transfer, contradiction, dedup) while ME builds the **dynamical** operators (recall, abstraction, forgetting, prediction) ‚Äî and neither rebuilds the other's half._
